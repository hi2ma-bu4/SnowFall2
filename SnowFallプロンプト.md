# 🛠️ SnowFall 言語：TS連携・中間コードシリアライゼーション実装指令

あなたは、独自言語「SnowFall」の開発チームにおける**システム・アーキテクト**です。現在完了している Rust 側のパース処理をベースに、仕様書 に基づく「TSからのソース受け渡し」と「超軽量中間コード（Bytecode-like Plain Text）」の生成ロジックを実装します。

## 1. 方針：ASCII範囲でのバイナリライク・フォーマット
中間コードは、仕様（1.1, 1.3）に基づき、以下の条件をすべて満たす必要があります。
* **最小サイズ:** 元のソースコードより小さく、不要な空白や識別子名を極限まで排除する。
* **低オーバーヘッド:** 実行段階（仕様ステップ6）でのデシリアライズを高速化するため、スタックマシン形式または固定長命令に近い構造を採用する。
* **ASCII互換:** 完全にバイナリ（Bytes）ではなく、プレーンテキストとして管理・転送可能なASCII範囲内に収める。

## 2. 実装タスク

### タスク C3: 中間コードのシリアライゼーション（Rust側）
ASTから独自の中間コードを生成するロジックを実装してください。
* **設計要件:** * 命令（OpCode）や型情報（type_id）を、1〜2文字のASCII文字（Base64等に近いエンコーディング）にマッピングし、バイナリパケットのような密度を実現する。
    * **デバッグ補助情報の分離:** `SnowFall`実行オプション（仕様1.3）が有効な場合のみ、マッピングデータを末尾に付与する構造にする。
* **提出コード:** `fn serialize_to_bytecode(ast: &Ast) -> String` のプロトタイプと、主要な命令のマッピングテーブル。

### タスク B1: Wasm インターフェースの構築（Rust/TS 境界）
TypeScriptからソースを受け取り、検証結果と中間コードを返すブリッジを構築してください。
* **設計要件:** * `wasm-bindgen` を用い、`compile(source: String, options: Config) -> CompileResult` をエクスポートする。
    * `CompileResult` は、中間コード（String）とエラーリスト（Vec<SnowFallError>）を保持すること（仕様1.3）。
* **提出コード:** Rust側の `pub fn compile` と、対応する TS 側の `SnowFallCompiler` クラス（仕様1.2）。

### タスク T1: TS側ラッパーと中間コード取得（TypeScript側）
仕様セクション2（動作フロー）のステップ3〜5を実装してください。
* **設計要件:** * ユーザーから受け取った文字列をWasmへ渡し、返却された中間コードをメモリ上に保持する。
    * エラー発生時は `SnowFallError` クラス（仕様1.3）のインスタンスとしてスローまたは返却する。

## 3. テスト駆動開発 (TDD) の実施
本フェーズの開発は、すべて **`npm run test`** による検証を前提とします。
* **検証項目:** 1.  **サイズ比較:** 生成された中間コードのバイト数が、元のソースコードの文字数以下であることをテストする。
    2.  **不変性:** 特定のソースコード（例：`Int a = 10;`）が常に同一のASCII中間コードを出力することを確認する。
    3.  **エラー精度:** 不正なソースを与えた際、`line` / `column` が正しく Rust から TS へ伝播していることをテストする（仕様1.3）。

## 4. 思考への問い
