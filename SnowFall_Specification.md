# SnowFall 言語 仕様書

SnowFallは、TypeScriptの厳密な型付けとPythonの柔軟性を融合させた、コンパイラ方式の独自言語です。

## 1. システム・開発環境仕様

### 1.1. 開発構成

| 項目               | 詳細                                                                         | 補足                                                                            |
| :----------------- | :--------------------------------------------------------------------------- | :------------------------------------------------------------------------------ |
| **開発言語**       | Rust (WebAssembly) および TypeScript                                         | 根幹・解析はRust (Wasm) 側で実装し、JS/TSの実行を減らすことで速度を確保します。 |
| **実行環境**       | Webブラウザ（クライアントサイド完結）                                        | すべての実行（検証、コンパイル、実行）はWeb上で動作します。                     |
| **動作段階**       | 検証 (Static Analysis)、コンパイル (Compilation)、実行 (Execution) の3段階。 | 各機能（`if`, `for`, `while` など）は個別のファイルに分離します。               |
| **コンパイル結果** | 文字化けしないプレーンテキスト形式。                                         | ASCII範囲であれば圧縮を許容します。                                             |

### 1.2. モジュール設計と独立性

* **機能独立性:** 検証&コンパイル機能と、実行機能は**それぞれ独立して動作**するようにファイルを分離します。
* **共通利用:** `util`などの共通利用モジュールは、TreeShakingを考慮して許容します。
* **言語要素の分離:** 言語機能（`Int`, `String`などの型、`if`, `for`などの構文）はそれぞれ個別にファイルに分割し、依存度を下げ、**拡張を容易**にします。
* **クラス構造:** 検証、コンパイル、実行の各機能は**クラス**で作成し、**コンストラクタ**で初期設定（オプション）を設定可能とします。

### 1.3. エラー処理とデバッグ補助

* **エラー情報:** コンパイルエラーおよび実行時エラーでは、**何行目の何文字目**で問題が発生したかを示す情報を表示可能とします。
* **エラークラス:** エラーは独自クラス（`SnowFallError`）で出力させます。
    * `SnowFallError`は、**エラー行数、文字数、スタックトレース**を含みます。
    * 他のエラークラスは`SnowFallError`を拡張して使用し、エラー定義は1ファイルにまとめます。
* **デバッグオプション:** コンパイル結果のデータサイズが増える場合は、`SnowFall`実行コンストラクタオプションで**デバッグ補助情報**の出力を変更できるようにします。

### 1.4. 開発規約

* **ドキュメント:** JSDocやRustのドキュメンテーションコメントは、**すべての関数、機能**について**日本語**で記載します。

## 2. 🚀 動作フロー（Rust/TS連携）

SnowFallの実行は、以下の手順でクライアントサイドで完結します。

| ステップ | 担当     | 処理内容                                                                                                             |
| :------- | :------- | :------------------------------------------------------------------------------------------------------------------- |
| **1**    | ユーザー | SnowFall言語のソースを作成します。                                                                                   |
| **2**    | TS       | ソースコードの**文字列**をRust (Wasm) に委託します。                                                                 |
| **3**    | Rust     | **静的検証**と**コンパイル**を実施し、**検証結果（エラー/警告リスト）**と**中間コード**をTSに返却します。            |
| **4**    | TS       | 返却された中間コードを取得します（エラーがあればユーザーに結果を提供）。                                             |
| **5**    | TS       | 中間コードをRust (Wasm) に委託します。                                                                               |
| **6**    | Rust     | 中間コードを解析し、実行します。必要に応じてTSと連携します（外部ライブラリなどの呼び出し、ユーザー拡張機能の実行）。 |
| **7**    | TS       | Rust実行結果をユーザーに返却します（必要に応じてRustとの対話を担います）。                                           |
| **8**    | ユーザー | 最終結果を受け取ります。                                                                                             |

## 3. 📝 言語仕様

### 3.1. 基本構文と型

* **基本構文:** Cライクの `{}` を使用した言語です。
    * `{}` はスコープの境目とします。
* **変数宣言:** `(型名) (変数名) = (値);` の形式で記載します。
    * 例: `Int foo = 0;`
* **型の種類:**
    * **プリミティブ型:** `Int`, `Long`, `Float`, `Char`, `String`, `Boolean` など。
    * **オブジェクト型:** `Function`, `Sub`, `Class`, `Array`, `Tuple`, `Dictionary`, `Object` など。
* **プロトタイプベース:** すべての型はJavaScriptと同様の構造（`__proto__`）を持つ**プロトタイプベース**で動作します。
    * すべての機能（型、クラス、関数）はSnowFallオブジェクトで作成されており、`String.__proto__.foo = foo;` のように**プロトタイプから機能を直接編集可能**です。
* **Object型:** `Object`型はすべての型を内包します（例: `String`は内部的に `Char[]` のように表現されます）。

### 3.2. 演算子と制御構文

#### 3.2.1. 等号・比較演算子

| 演算子 | 意味             | 補足                                         |
| :----- | :--------------- | :------------------------------------------- |
| `=`    | 代入             | すべての場合で代入に使用されます。           |
| `==`   | 型を無視した比較 | **暗黙の型変換**を伴う比較です。             |
| `===`  | 型比較も含む比較 | 型と値の両方を比較します（TypeScript方式）。 |

* **暗黙の型変換の優先順位:** `==`での型変換は、優先順位が高い型に合わせます。
    * **優先度順:** `String` > `Float` > `Int` > `Char` > `Boolean`

#### 3.2.2. 論理演算子

| 演算子        | 意味             | 評価方法                                                 |
| :------------ | :--------------- | :------------------------------------------------------- |
| `&&` / `\|\|` | 論理AND / 論理OR | **短絡評価**で計算します。                               |
| `and` / `or`  | 論理AND / 論理OR | **両端を実行**するものとします（短絡評価を行いません）。 |

#### 3.2.3. 特殊な計算

* **文字列連結:** 文字列と整数の乗算で文字列連結を行います。
    * 例: `(stringB) = (stringA) * (int)` は、`stringA`を`int`回連結した内容を`stringB`として返却します。

#### 3.2.4. 制御構文

* **If文:** TypeScript方式で比較します。
    * 例: `if (foo === bar) { /* ... */ }`
* **For文:** 標準ループとforeachに対応します。
    * 標準ループ: `for (Int i=0;i<10;i++) { /* ... */ }`
    * ForEachループ: `for (String key in (dictA)) { /* ... */ }`

### 3.3. クラスとオーバーライド

* **コンストラクタ:** クラスのコンストラクタは `__init__()` です。
* **演算子オーバーロード:** `__add__()` などを使用することで、足し算などの演算をオーバーライド可能です。
* **オーバーライドとラッパー:**
    * オーバーライドするメソッドの前には `@override` を記載します。
    * その他のラッパー（デコレーター）も `@foo` 形式で定義します。

### 3.4. 型変換と数値リテラル

* **型変換（キャスト）:** C言語形式のキャストが可能です。
    * 例: `(int)"0x13"`
* **型変換（メソッド）:** 標準ライブラリのメソッドも利用可能です。
    * 例: `Integer.parse("0b101")`
* **数値リテラル:** 以下の基数で指定可能です。また、浮動小数点数の特殊な形式にも対応します。
    * 10進数: `99`
    * 16進数: `0xff`
    * 2進数: `0b11`
    * 特殊な小数: `0xf.f` (16進数ベースの小数表現)

### 3.5. 再帰制御

* **再帰:** `function` および `sub` は再帰可能です。
* **再帰上限:** 再帰回数の上限は、`SnowFall`実行コンストラクタオプションで個別に指定可能です。
    * 初期値は **10,000回** です。
    * 上限チェックは **Rust側** で行います。
